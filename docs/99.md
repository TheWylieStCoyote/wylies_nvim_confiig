## Architecture

99 is a thin Neovim wrapper. It does **not** talk to LLM APIs directly â€” it shells out to a separately-installed CLI tool (the "provider") which handles auth and model dispatch. Your config picks which CLI to drive.

## Providers

| Provider class | CLI binary | Default model | Auth |
|---|---|---|---|
| `OpenCodeProvider` | `opencode` | `opencode/claude-sonnet-4-5` | `opencode auth login` |
| `ClaudeCodeProvider` *(active)* | `claude` | `claude-sonnet-4-5` | Claude Code session |

| `GeminiCLIProvider` | `gemini` | `auto` | `gemini auth` |

> **Why not OpenCode by default on this machine?** `opencode auth list` only shows GitHub Copilot OAuth, which doesn't expose `claude-sonnet-4-5`. ClaudeCodeProvider is the lower-friction path here.

## Operations Cheat Sheet

### Search â€” read-only investigation

Best for: "where is X handled?", "list all callers of Y", "find every TODO related to Z".

1. `<leader>9s` opens a prompt buffer.
2. Type the question, submit.
3. Results land in the **quickfix list** â€” navigate with `:cnext`/`:cprev` or `]q`/`[q`.

### Visual â€” targeted replacement

Best for: filling in a stub, refactoring a block, expanding a comment into code.

1. Visual-select (`v`/`V`/`<C-v>`).
2. `<leader>9v`, prompt with the instruction.
3. Selection is replaced with the LLM output.

> **Footgun:** the keymap operates on your *last* visual selection. The binding is visual-only on purpose â€” don't trigger from normal mode without a fresh selection.

### Vibe â€” agentic multi-file edits

Best for: "add a feature", "create module + tests", "wire up a new endpoint".

1. `<leader>9b` from any line. A virtual-text marker `[99 vibe pendingâ€¦]` appears at end-of-line so you can find where the request was launched from in long sessions.
2. Prompt window opens. Type the instruction.
3. Provider does the work â€” creating/editing files. Quickfix lists what changed.
4. Marker clears on the **first cursor move in the originating buffer**.

## Configuration

Located in `lua/plugins/99.lua`.

```lua
_99.setup({
  provider = _99.Providers.ClaudeCodeProvider,
  logger = {
    level = _99.INFO,
    path = "/tmp/" .. basename .. ".99.log",
    print_on_error = true,
  },
  tmp_dir = "./tmp",
  md_files = { "AGENT.md" },
  completion = { source = "blink" },
})
```

| Setting | Why |
|---|---|
| `provider = ClaudeCodeProvider` | Use the already-authed `claude` CLI |
| `tmp_dir = "./tmp"` | Must live inside CWD or the underlying CLI refuses file ops |
| `md_files = { "AGENT.md" }` | Auto-pulls AGENT.md from project root upward when present |
| `completion.source = "blink"` | Matches LazyVim's blink.cmp; needs `saghen/blink.compat` for full integration |
| `logger.path` | Per-project log file at `/tmp/<project>.99.log` |

### Custom: line-anchored vibe marker

`lua/plugins/99.lua` defines a `vibe_with_line_marker` wrapper instead of calling `_99.vibe()` directly. It places a virtual-text extmark at end-of-line on the request's originating line, scoped to a one-shot `CursorMoved`/`CursorMovedI` autocmd in the originating buffer.

This is layered on top of 99's built-in top-right throbber â€” both are visible during a request. The wrapper uses only public Neovim APIs so it survives 99 plugin updates.

## Workflow Tips

- Drop an `AGENT.md` in your project root with project-specific context. 99 auto-injects it on every request.
- Telescope model picker (`<leader>9m`) only persists for the current nvim session â€” restart reverts to your `setup()` defaults.
- For "add code" tasks, prefer **vibe** (multi-file) for new features and **visual** (single-file) for filling in stubs.
- Log file path is shown in your `setup()` config â€” `tail -f /tmp/<project>.99.log` while iterating on prompts to watch real provider traffic.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Throbber spins forever | Provider CLI hung or auth failed | `<leader>9x` to kill, then `<leader>9l` for logs |
| Empty quickfix after search/vibe | Provider responded but didn't produce results â€” usually too vague a prompt | Be more specific about file paths and scope |
| Files created outside CWD | Permission/cwd bug â€” should not happen with `tmp_dir = "./tmp"` | `<leader>9x` immediately and check log |
| `[99] blink.compat is required` warning | `completion.source = "blink"` needs `saghen/blink.compat` shim | Either install `saghen/blink.compat` or change source to `"native"` |
| `<leader>9m` does nothing | Telescope not loaded yet | Verified working; if regressed, run `:lua require("99.extensions.telescope").select_model()` to see the actual error |
| Marker stuck after vibe completes | You haven't moved the cursor in the originating buffer yet | Move cursor; or re-trigger `<leader>9b` to replace the stale marker |
| No results at all on first try | OpenCode authed only to Copilot, default model unavailable | This config uses ClaudeCodeProvider; if you switched back to OpenCode, run `opencode auth login` and add Anthropic |
