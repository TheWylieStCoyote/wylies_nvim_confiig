# C/C++ Development

> LSP: clangd | Formatter: clang-format | Linter: clang-tidy | Debugger: codelldb

## Quick Reference

| Component | Tool |
|-----------|------|
| LSP | clangd |
| Formatter | clang-format |
| Linter | clang-tidy (via clangd) |
| Debugger | codelldb |
| TreeSitter | c, cpp, cmake, make, meson |

## Features

- Background indexing
- Clang-tidy integration
- Header insertion (IWYU)
- Detailed completions with placeholders
- Inlay hints
- AST visualization (via clangd_extensions)
- Type hierarchy
- Memory usage analysis
- Switch between source/header files

## Configuration

### compile_commands.json

For best results, generate `compile_commands.json`:

**CMake:**
```bash
cmake -DCMAKE_EXPORT_COMPILE_COMMANDS=ON ..
```

**Bear (for Make):**
```bash
bear -- make
```

### .clang-format

Create a `.clang-format` file in your project root:
```yaml
BasedOnStyle: LLVM
IndentWidth: 4
ColumnLimit: 100
```

### .clang-tidy

Create a `.clang-tidy` file:
```yaml
Checks: '*,-fuchsia-*,-google-*,-llvm-*'
WarningsAsErrors: ''
```

### .clangd

A `.clangd` file lives at your project root and lets you configure how clangd analyzes
your code. It is **not** part of the build system — it is read directly by clangd. It
layers on top of `compile_commands.json` if one exists.

**Priority order clangd uses:**
1. `compile_commands.json` — most accurate, generated by your build system
2. `.clangd` — project-level overrides and additions
3. `compile_flags.txt` — flat list of flags, simplest fallback
4. Built-in defaults + `--query-driver` results

#### Bare minimum

```yaml
CompileFlags:
  Add: ["-std=c++17"]
```

#### Modern C++ project

```yaml
CompileFlags:
  Add:
    - "-std=c++23"
    - "-Wall"
    - "-Wextra"
    - "-Wpedantic"
```

#### Embedded / cross-compile (ARM bare metal)

```yaml
CompileFlags:
  Add:
    - "-std=c++17"
    - "-mcpu=cortex-m4"
    - "-mthumb"
    - "-mfpu=fpv4-sp-d16"
    - "-mfloat-abi=hard"
    - "-DSTM32F4"
    - "-ffreestanding"
    - "-fno-exceptions"
    - "-fno-rtti"
  Remove:
    - "-march=native"   # not valid for cross-compilers
Compiler: arm-none-eabi-g++
```

#### CUDA project (without nvcc as the compiler)

```yaml
CompileFlags:
  Add:
    - "-std=c++17"
    - "-x"
    - "cuda"
    - "--cuda-gpu-arch=sm_75"
    - "-I/usr/local/cuda/include"
    - "-D__CUDACC__"
```

#### Project with noisy vendor flags clangd doesn't understand

```yaml
CompileFlags:
  Add:
    - "-std=c++17"
    - "-DPLATFORM_LINUX"
  Remove:
    - "-fanalyzer"             # GCC-only, clangd chokes on it
    - "-Wno-psabi"             # GCC-only warning suppression
    - "-fmacro-prefix-map=.*"  # causes clangd parse errors
```

#### ROS2 / robotics project

```yaml
CompileFlags:
  Add:
    - "-std=c++17"
    - "-DROS_PACKAGE_NAME=\"my_package\""
    - "-I/opt/ros/humble/include"
    - "-I/opt/ros/humble/include/rclcpp"
Diagnostics:
  Suppress:
    - "pp_including_mainfile_in_preamble"  # common with ROS headers
```

#### Suppress specific noisy diagnostics

```yaml
CompileFlags:
  Add: ["-std=c++17"]
Diagnostics:
  Suppress:
    - "unused-includes"          # if IWYU is too aggressive
    - "pp_pragma_once_in_main"
  ClangTidy:
    Add:
      - "modernize-*"
      - "performance-*"
    Remove:
      - "modernize-use-trailing-return-type"  # too opinionated
```

#### Multi-root project (different flags per subdirectory)

Place a `.clangd` at the repo root, then another inside the subdirectory that needs
different flags. clangd merges them, with the closer file taking precedence.

Root `.clangd`:
```yaml
CompileFlags:
  Add: ["-std=c++20"]
```

`src/legacy/.clangd`:
```yaml
CompileFlags:
  Add: ["-std=c++11"]   # legacy code can't use newer standard
  Remove: ["-std=c++20"]
```

#### Full-featured example

```yaml
CompileFlags:
  Add:
    - "-std=c++20"
    - "-Wall"
    - "-Wextra"
    - "-Isrc"
    - "-Iinclude"
    - "-Ithird_party/fmt/include"
    - "-DDEBUG_BUILD"
    - "-DPROJECT_VERSION=\"1.0.0\""
  Remove:
    - "-Wno-psabi"
    - "-fanalyzer"

Diagnostics:
  UnusedIncludes: Strict
  ClangTidy:
    Add:
      - "modernize-*"
      - "performance-*"
      - "bugprone-*"
      - "cppcoreguidelines-*"
    Remove:
      - "modernize-use-trailing-return-type"
      - "cppcoreguidelines-avoid-magic-numbers"
    CheckOptions:
      modernize-use-default-member-init.UseAssignment: true

Index:
  Background: Build    # index in the background

InlayHints:
  Enabled: true
  ParameterNames: true
  DeducedTypes: true
```

#### Key fields reference

| Field | Purpose |
|---|---|
| `CompileFlags.Add` | Extra flags passed to clang |
| `CompileFlags.Remove` | Strip flags that confuse clangd |
| `Compiler` | Override which compiler binary clangd queries |
| `Diagnostics.Suppress` | Silence specific diagnostic codes |
| `Diagnostics.ClangTidy.Add/Remove` | Control which clang-tidy checks run |
| `InlayHints` | Toggle/configure inline type and param hints |
| `Index.Background` | Whether to index in the background (`Build`/`Skip`) |

## Usage Examples

### Quick Build & Run
1. Open a C++ file
2. `<leader>cg` - Generate CMake
3. `<leader>cb` - Build
4. `<leader>cr` - Run

### Debug Session
1. `<leader>cd` - Start CMake Debug
2. `<F9>` - Set breakpoints
3. `<F5>` - Start debugging
4. `<F10>` / `<F11>` - Step through code
